# Chat Processing Pipeline Demo

Полная цепочка обработки чата менеджера с клиентом: от сырых сообщений до оценки качества.

## Файлы

| Файл | Что внутри |
|------|-----------|
| [`chat_processing_demo.ipynb`](chat_processing_demo.ipynb) | Jupyter notebook — пошаговая демонстрация с реальным вызовом DeepSeek API через curl |
| [`chat_pipeline_flow.html`](chat_pipeline_flow.html) | Визуальная блок-схема всей цепочки обработки |

## Цепочка обработки

```
CRM присылает сообщения
        |
   Дедупликация
        |
   Группировка в диалоги (менеджер + клиент + мессенджер)
        |
   Фильтрация (>= 4 сообщений, не бот)
        |
   +----+----+
   |         |
Код       ИИ (DeepSeek)
(время     (смысл текста)
ответа)        |
   |      +---+---+
   |      |   |   |
   |   Чеклист Теги Данные
   |      |   |   |
   +------+---+---+
        |
   Валидация (retry при ошибках)
        |
   Callback JSON в CRM
```

## Ключевые особенности

- **`evidence_msg_ids`** — каждый результат содержит массив ID сообщений-доказательств (CRM может подсветить конкретные сообщения)
- **Два типа проверок**: по времени (код) и по смыслу (ИИ) — разные ветки
- **Retry** — при ошибке сети, битом JSON или невалидных ID — автоматический повторный запрос (до 3 раз)
- **curl** — вызов ИИ через обычный HTTP POST (аналог `curl_exec` в PHP)
